## 003: PDF 页面渲染（PyMuPDF 集成）

**Category:** New | **Priority:** Critical | **Status:** Completed | **Started:** 2026-03-23

### 描述

集成 PyMuPDF，将 PDF 每页渲染为高 DPI 图像，并构造 PageImage 数据对象。

### 设计要点

- 参考 DD §2.1 Parser Stage
- 实现 `stages/parser.py` 的核心逻辑
- `fitz.open` 打开 PDF → `Page.get_pixmap(dpi=N)` 渲染 → 转 numpy array
- 构造 `PageImage` dataclass（frozen）
- v0.1.0 不需要提取嵌入资源（EmbeddedResource 留到 v0.3.0）

### 2. 影响范围

**需要修改的文件**:
- `src/noteeditor/stages/parser.py` — 替换骨架，实现 PDF 解析逻辑

**需要创建的文件**:
- `tests/unit/test_parser.py` — 解析器单元测试

**相关现有文件**（参考模式）:
- `src/noteeditor/models/page.py` — PageImage, PageMetadata dataclass
- `src/noteeditor/errors.py` — InputError 异常
- `docs/DD.md §2.1` — Parser Stage 详细设计

### 3. 技术方案

**依赖**:
- pymupdf (fitz): PDF 解析与渲染（已安装）
- numpy: 图像数组（已安装）

**核心函数**:
```python
def parse_pdf(input_path: Path, dpi: int) -> tuple[PageImage, ...]:
    """解析 PDF 文件，将每页渲染为 PageImage。"""

def render_page(page: fitz.Page, page_number: int, total_pages: int, dpi: int) -> PageImage:
    """渲染单个 PDF 页面为 PageImage。"""

def pixmap_to_numpy(pixmap: fitz.Pixmap) -> np.ndarray:
    """将 PyMuPDF Pixmap 转为 RGB numpy 数组。"""
```

**渲染流程**:
1. `fitz.open(path)` 打开 PDF
2. 遍历每页: `page.get_pixmap(dpi=N)` → Pixmap
3. Pixmap → numpy: `np.frombuffer(samples, dtype=np.uint8).reshape(h, w, n)`
4. RGBA → RGB 转换（Pixmap alpha 通道处理）
5. 构造 PageImage (frozen dataclass, 0-based page_number)

**错误处理**:
- PDF 无法打开: 抛出 `InputError`
- 单页渲染失败: 记录警告, 跳过该页, 继续处理

### 4. 接口契约

```python
# stages/parser.py
def parse_pdf(input_path: Path, dpi: int) -> tuple[PageImage, ...]:
    """解析 PDF，返回每页的 PageImage。"""
    # Raises: InputError (无法打开 PDF)

def render_page(page: fitz.Page, page_number: int, total_pages: int, dpi: int) -> PageImage:
    """渲染单页为 PageImage。"""
```

### 5. 实现步骤

1. **实现 pixmap_to_numpy()** → Pixmap 转 RGB numpy 数组
2. **实现 render_page()** → 单页渲染 + PageImage 构造
3. **实现 parse_pdf()** → PDF 打开 + 遍历 + 错误隔离
4. **编写测试** → 正常渲染、DPI 缩放、单页失败、空 PDF、损坏 PDF

### 6. 验收标准

- [ ] 输入 NotebookLM PDF，输出正确数量的 PageImage
- [ ] 默认 300 DPI 渲染，`--dpi 600` 时分辨率翻倍

## 004: 宽高比检测与 PPTX 尺寸匹配

**Category:** New | **Priority:** High | **Status:** Completed

### 描述

检测 PDF 页面宽高比，据此设置 PPTX 幻灯片尺寸，确保输出 PPTX 的比例与输入一致。

### 设计要点

- 从 PageImage 的 width_px / height_px 计算宽高比
- NotebookLM 导出为 16:9，映射为 slide_width=13.333in, slide_height=7.5in
- 支持其他常见比例（4:3 → 10in × 7.5in）
- 宽高比写入 PipelineConfig 供 Builder 使用

### 验收标准

- 16:9 PDF 输出 13.333in × 7.5in 的 PPTX
- 4:3 PDF 输出 10in × 7.5in 的 PPTX
- 非标准比例时取最接近的标准比例并打印警告
