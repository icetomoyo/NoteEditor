## 002: CLI 入口与参数解析

**Category:** New | **Priority:** Critical | **Status:** InProgress

### 描述

实现 CLI 入口，支持 v0.1.0 所需的基本参数：输入 PDF 路径、输出 PPTX 路径、DPI 设置。

### 设计要点

- 参考 HLD §5 CLI 接口设计
- v0.1.0 仅支持部分参数：`input`, `-o/--output`, `--dpi`
- 其余参数预留但不实现（解析后忽略或提示 "not yet supported"）
- 使用 click 框架
- 入口点在 pyproject.toml 中注册为 console_scripts

### 2. 影响范围

**需要修改的文件**:
- `src/noteeditor/cli.py` — 实现 CLI 完整逻辑

**需要创建的文件**:
- `src/noteeditor/errors.py` — 自定义异常类

**相关现有文件**（参考模式）:
- `src/noteeditor/models/page.py` — frozen dataclass 模式
- `docs/HLD.md §5` — CLI 接口设计规范

### 3. 技术方案

**依赖**:
- click: CLI 框架（已安装）

**输入验证**:
- 文件存在性: click.Path(exists=True) 处理
- PDF 格式: 先检查扩展名 .pdf，再检查 magic bytes `%PDF-`
- DPI 范围: 72-1200，默认 300

**输出路径推导**:
- 无 `-o` 时: 输入文件同目录，同文件名，后缀改 .pptx
- 有 `-o` 时: 使用指定路径
- 父目录不存在时: 自动创建 (os.makedirs)

**错误处理**:
- InputError: 文件不存在、非 PDF、不可读
- 使用 click.echo + sys.exit(1) 输出友好错误

### 4. 接口契约

```python
# errors.py
class InputError(Exception): ...
class OutputError(Exception): ...

# cli.py
def main(input_pdf: str, output: str | None, dpi: int) -> None:
    """Convert a NotebookLM PDF to PPTX."""
```

**CLI 参数**:
| 参数 | 缩写 | 类型 | 默认值 | 描述 |
|------|------|------|--------|------|
| `input_pdf` | | argument | 必填 | 输入 PDF 路径 |
| `--output` | `-o` | option | 自动推导 | 输出 PPTX 路径 |
| `--dpi` | | option | 300 | 渲染 DPI |

### 5. 实现步骤

1. **创建 errors.py** → InputError, OutputError
2. **实现 validate_input()** → 文件存在 + PDF 格式校验
3. **实现 resolve_output_path()** → 自动推导输出路径
4. **实现 validate_dpi()** → DPI 范围校验
5. **更新 main()** → 整合验证逻辑

### 6. 验收标准

- [x] `noteeditor input.pdf` 生成默认路径的 .pptx 文件
- [x] `noteeditor input.pdf -o output.pptx` 指定输出路径

- [x] 无输入参数时显示 usage 并退出

## 003: PDF 页面渲染（PyMuPDF 集成）

**Category:** New | **Priority:** Critical | **Status:** Planned

### 描述

集成 PyMuPDF，将 PDF 每页渲染为高 DPI 图像，并构造 PageImage 数据对象。

### 设计要点

- 参考 DD §2.1 Parser Stage
- 实现 `stages/parser.py` 的核心逻辑
- `fitz.open` 打开 PDF → `Page.get_pixmap(dpi=N)` 渲染 → 转 numpy array
- 构造 `PageImage` dataclass（frozen）
- v0.1.0 不需要提取嵌入资源（EmbeddedResource 留到 v0.3.0）

### 验收标准

- 输入 NotebookLM PDF，输出正确数量的 PageImage
- 默认 300 DPI 渲染，`--dpi 600` 时分辨率翻倍
- PageImage.width_px / PageImage.height_px 与 DPI 和 PDF 页面尺寸一致
- 对非 PDF 文件抛出 InputError

## 004: 宽高比检测与 PPTX 尺寸匹配

**Category:** New | **Priority:** High | **Status:** Planned

### 描述

检测 PDF 页面宽高比，据此设置 PPTX 幻灯片尺寸，确保输出 PPTX 的比例与输入一致。

### 设计要点

- 从 PageImage 的 width_px / height_px 计算宽高比
- NotebookLM 导出为 16:9，映射为 slide_width=13.333in, slide_height=7.5in
- 支持其他常见比例（4:3 → 10in × 7.5in）
- 宽高比写入 PipelineConfig 供 Builder 使用

### 验收标准

- 16:9 PDF 输出 13.333in × 7.5in 的 PPTX
- 4:3 PDF 输出 10in × 7.5in 的 PPTX
- 非标准比例时取最接近的标准比例并打印警告

## 005: Pipeline Orchestrator（串行调度）

**Category:** New | **Priority:** Critical | **Status:** Planned

### 描述

实现简化版 Pipeline Orchestrator，串行调度 Parser → Builder，完成多页 PDF 的遍历处理和单页错误隔离。

### 设计要点

- 参考 HLD §4 Pipeline Orchestrator
- 实现 `pipeline.py` 核心逻辑（v0.1.0 简化版）
- v0.1.0 调度流程：
  1. 加载 PipelineConfig
  2. 调用 Parser 获取 PageImage 列表
  3. 遍历每页：调用 Builder 将 PageImage 写入 PPTX
  4. 单页失败时捕获异常，记录错误，跳过该页继续
- v0.1.0 不需要：并行调度、断点续传、checkpoint
- 错误隔离：单页异常不中断整体流程

### 验收标准

- 3 页 PDF 全部成功时，输出 3 页 PPTX
- 第 2 页渲染失败时，输出 2 页 PPTX（跳过失败页），并在 stderr 打印错误信息
- 全部页面失败时，输出 0 页空 PPTX 并报错
- Orchestrator 作为独立模块，不依赖 CLI 或 Builder 的具体实现
