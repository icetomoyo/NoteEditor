## 009: 布局检测（PP-DocLayout-V3）

**Category:** New | **Priority:** Critical | **Status:** Planned

### 描述

集成 PP-DocLayout-V3 文档布局检测模型，对每页进行语义区域检测，识别标题、正文、图片、表格、页眉、页脚等区域，输出边界框坐标和语义标签。

### 设计要点

- 参考 DD §2.2 Layout Stage
- PP-DocLayout-V3（Baidu，Apache 2.0）— GLM-OCR 官方也在使用此模型作为布局检测骨干
- 输入：PageImage.image (H, W, 3) RGB
- 输出：LayoutResult（每页的 LayoutRegion 列表）
- 数据模型已定义：`models/layout.py`（LayoutRegion, LayoutResult, BoundingBox, RegionLabel）
- 运行时：onnxruntime CPU/GPU 自适应

### 2. 影响范围

**需要修改的文件**:
- `src/noteeditor/stages/layout.py` — 替换骨架，实现布局检测逻辑

**需要创建的文件**:
- `src/noteeditor/infra/model_manager.py` — 模型下载、验证、加载
- `tests/unit/test_layout.py` — 布局检测单元测试

**相关现有文件**（参考模式）:
- `src/noteeditor/models/layout.py` — LayoutRegion, LayoutResult, RegionLabel dataclass
- `src/noteeditor/stages/parser.py` — Stage 模式参考
- `docs/DD.md §2.2` — Layout Stage 详细设计

### 3. 技术方案

**核心函数**:
```python

    """Run layout detection on a single page image."""

    """Filter out low-confidence regions."""
```

**推理流程**:
1. 将 `PageImage.image` 转为模型输入格式（resize + normalize）
2. 运行 PP-DocLayout-V3 推理
3. 解析输出：`[(bbox, label, confidence), ...]`
4. 映射模型标签 → `RegionLabel` 枚举
5. 过滤低置信度区域（`confidence < 0.5`）
6. 构造 `LayoutResult` (frozen dataclass)

**标签映射**:
- PP-DocLayout-V3 原始标签 → `RegionLabel` 枚举的映射表作为模块内部常量维护
- 无法映射的标签使用 `RegionLabel.UNKNOWN`

## 010: OCR 文字提取（GLM-OCR）

**Category:** New | **Priority:** Critical | **Status:** Planned

### 描述

集成 GLM-OCR 文字识别模型，对布局检测到的文字区域进行 OCR 识别，提取文本内容。支持中英文混合和公式识别。

### 设计要点

- 参考 DD §2.3 OCR Stage
- GLM-OCR（智谱，0.9B 参数，Apache 2.0）— OmniDocBench v1.5 得分 94.62
- 输入：PageImage + LayoutResult（过滤文字类型区域）
- 输出：`list[OCRResult]`
- 数据模型已定义：`models/content.py`（OCRResult）
- 文字区域裁切时 padding=10px 防止文字截断

### 2. 影响范围

**需要修改的文件**:
- `src/noteeditor/stages/ocr.py` — 替换骨架，实现 OCR 逻辑
- `src/noteeditor/infra/model_manager.py` — 添加 OCR 模型加载

**需要创建的文件**:
- `tests/unit/test_ocr.py` — OCR 单元测试

**相关现有文件**（参考模式）:
- `src/noteeditor/models/layout.py` — RegionLabel 枚举
- `src/noteeditor/models/content.py` — OCRResult dataclass
- `docs/DD.md §2.3` — OCR Stage 详细设计

### 3. 技术方案

**依赖**:
- onnxruntime: 模型推理（已安装）
- numpy: 图像裁切和处理（已安装）

**文字区域过滤**:
```python
_TEXT_LABELS = {RegionLabel.TITLE, RegionLabel.BODY_TEXT, RegionLabel.EQUATION, RegionLabel.CODE_BLOCK}
```

**核心函数**:
```python
# stages/ocr.py
def extract_text(
    page_image: PageImage,
    layout_result: LayoutResult,
    session: InferenceSession,
) -> tuple[OCRResult, ...]:
    """Run OCR on text-type regions detected by layout."""

def _crop_region(page_image: PageImage, bbox: BoundingBox, padding: int = 10) -> np.ndarray:
    """Crop a region from the page image with padding."""
```

**推理流程**:
1. 从 `LayoutResult` 中过滤文字类型区域
2. 对每个文字区域：
   a. 根据 `bbox` 从 `PageImage.image` 裁切区域图（+10px padding）
   b. 运行 GLM-OCR 推理：输入裁切图 → 输出文本 + is_formula + LaTeX
   c. 构造 `OCRResult` (frozen dataclass)
3. 返回 `OCRResult` 列表

**公式处理**（v0.2.0 MVP 简化）:
- GLM-OCR 标记为公式的文本：`is_formula=True`，存储 `formula_latex`
- v0.2.0 简化处理：公式渲染为文本框，LaTeX 源码直接放入文本内容
- 完整 LaTeX 渲染（matplotlib → PNG → 嵌入）留到后续版本

### 4. 接口契约

## 011: 可编辑文本框放置

**Category:** New | **Priority:** Critical | **Status:** Planned

### 描述

扩展 PPTX Builder，在可编辑模式下将 OCR 识别的文字以原生文本框放置在幻灯片上。v0.2.0 MVP 策略：原始截图作为幻灯片背景 + 白色填充文本框覆盖原始文字。

### 设计要点

- 参考 DD §2.7 Builder Stage（可编辑模式部分）
- v0.2.0 MVP 策略：
  - 原始截图作为幻灯片背景（`full_page_image`）
  - OCR 识别的文字以白色填充的文本框覆盖原始文字
  - 使用默认系统字体（Arial）
  - 字号从 bbox 高度估算：`font_size_pt = bbox_height_px * (72 / dpi) * 0.8`
  - 图片区域直接放置（不提取，保留在背景中）
  - 失败页面回退到纯截图模式
- 数据模型已定义：`models/slide.py`（TextBlock, ImageBlock, SlideContent）
- v0.2.0 跳过：字体匹配（FontMatch 留到 v0.4.0）、LaTeX 渲染、图片提取、背景重建

### 3. 技术方案

**依赖**:
- python-pptx: PPTX 创建（已安装）

**核心函数**:

**Assemble 阶段（OCRResult → TextBlock → SlideContent）**:
```python

    page_image: PageImage,
    layout_result: LayoutResult,
    ocr_results: tuple[OCRResult, ...],

```

**文本框放置策略**:
1. 位置：`bbox` 坐标转换为 EMU（1 inch = 914400 EMU）
   ```
   left_emu = bbox.x * (914400 / dpi)
   top_emu = bbox.y * (914400 / dpi)
   width_emu = bbox.width * (914400 / dpi)
   height_emu = bbox.height * (914400 / dpi)
   ```
2. 填充：白色背景（`fill.solid()`，`fill.fore_color.rgb = RGBColor(0xFF, 0xFF, 0xFF)`）
3. 字体：Arial（默认系统字体）
4. 字号：从 bbox 高度估算
5. 对齐：根据 RegionLabel（TITLE=居中，BODY_TEXT=左对齐）

**失败回退**:
- Layout 检测失败：整页回退到截图模式
- OCR 失败（某区域）：该区域跳过文本框，保留背景中的原始文字

## 012: Pipeline 多阶段扩展

**Category:** Internal | **Priority:** Critical | **Status:** Planned

### 描述

扩展 Pipeline Orchestrator，从 v0.1.0 的 Parser → Builder 两阶段串行调度，升级为 Parser → Layout → OCR → Assemble → Builder 五阶段调度。添加 `mode` 配置支持视觉保真/可编辑两种模式。

### 设计要点

- 参考 DD §1.5 PipelineConfig 和 HLD §4
- v0.2.0 调度流程：
  1. Parser：PDF → PageImage 列表
  2. Layout：每页运行布局检测 → LayoutResult
  3. OCR：每页运行文字识别 → OCRResult
  4. Assemble：组合 LayoutResult + OCRResult → SlideContent
  5. Builder：SlideContent 列表 → PPTX
- 新增 `mode` 配置：`visual`（截图模式，v0.1.0 行为）/ `editable`（可编辑模式，v0.2.0 新行为）
- 保持单页错误隔离：任意阶段某页失败，该页回退到截图模式

### 2. 影响范围

**需要修改的文件**:
- `src/noteeditor/pipeline.py` — 扩展为五阶段调度
- `src/noteeditor/infra/config.py` — PipelineConfig 添加 `mode` 字段
- `src/noteeditor/cli.py` — 添加 `--mode` CLI 选项
- `tests/unit/test_pipeline.py` — 更新 pipeline 测试

**需要创建的文件**:
- 无（使用现有模型定义）

**相关现有文件**（参考模式）:
- `src/noteeditor/pipeline.py` — 现有 v0.1.0 串行调度
- `src/noteeditor/infra/config.py` — PipelineConfig dataclass
- `docs/DD.md §1.5` — PipelineConfig 设计

### 3. 技术方案

**PipelineConfig 扩展**:
```python
@dataclass(frozen=True)
class PipelineConfig:
    input_path: Path
    output_path: Path
    dpi: int = 300
    verbose: bool = False
    mode: Literal["visual", "editable"] = "editable"  # v0.2.0 默认可编辑
    models_dir: Path = Path("~/.noteeditor/models").expanduser()
```

**CLI 新增选项**:
```
--mode visual|editable    输出模式（默认: editable）
```

**核心流程**:
```python
def run_pipeline(config: PipelineConfig) -> PipelineResult:
    # Stage 1: Parse
    pages = parse_pdf(config.input_path, config.dpi)

    # Stage 2-4: Per-page processing (layout, OCR, assemble)
    slides = []
    for page in pages:
        try:
            layout = detect_layout(page, model_mgr.get_layout_model())
            ocr_results = extract_text(page, layout, model_mgr.get_ocr_model())
            slide = assemble_slide(page, layout, ocr_results)
        except Exception:
            # Fallback: use screenshot mode for this page
            slide = SlideContent(
                page_number=page.page_number,
                background_image=None,
                full_page_image=page.image,
                text_blocks=(),
                image_blocks=(),
                status="fallback",
            )
        slides.append(slide)

    # Stage 5: Build
    if config.mode == "editable":
        build_editable_pptx(slides, config.output_path, config.dpi)
    else:
        build_pptx(tuple(p.image for p in pages), config.output_path)
```
