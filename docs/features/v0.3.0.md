## 013: 程序化背景重建

**Category:** New | **Priority:** High | **Status:** InProgress | **Started:** 2026-04-08

### 描述

实现 Background Extractor 阶段，从页面渲染图中移除文字区域，生成干净背景图像。替换 v0.2.0 的白色填充策略，使可编辑 PPTX 的背景更自然。支持简单（纯色/近似纯色）和渐变背景的自动检测与重建，复杂背景回退到白色填充。

### 设计要点

- 参考 DD §2.6 Background Extractor
- v0.3.0 策略：
  - 简单背景（std < 15）：用背景区域中值颜色填充文字区域
  - 渐变背景（15 ≤ std < 50）：按行线性插值填充
  - 复杂背景（std ≥ 50）：回退到白色填充（LaMA Inpainting 留到 v0.6.0 Feature 020）
- 输入：PageImage + LayoutResult
- 输出：`np.ndarray`（干净背景图像）
- 数据模型已有 `background_image` 字段（SlideContent，v0.2.0 未使用）

### 3. 技术方案

**依赖**: numpy, opencv-python-headless, pillow（已安装）

**核心函数**:
```python
def _create_text_mask(
    image_shape: tuple[int, int],
    regions: tuple[LayoutRegion, ...],
    text_labels: frozenset[RegionLabel],
) -> np.ndarray:
    """Create binary mask: 255=text regions, 0=background."""

def _classify_background(
    image: np.ndarray,
    mask: np.ndarray,
) -> Literal["simple", "gradient", "complex"]:
    """Classify background complexity from non-masked pixels."""

def _fill_simple(image: np.ndarray, mask: np.ndarray) -> np.ndarray:
    """Fill masked regions with median color of background."""

def _fill_gradient(image: np.ndarray, mask: np.ndarray) -> np.ndarray:
    """Fill masked regions with per-row linear interpolation."""

def _fill_fallback(image: np.ndarray, mask: np.ndarray) -> np.ndarray:
    """Fill masked regions with white (fallback for complex)."""

def extract_background(
    page_image: PageImage,
    layout_result: LayoutResult,
) -> np.ndarray:
    """Extract clean background from page image."""
```

**背景分类算法**:
1. 取 mask=0 区域（背景）的像素
2. 计算颜色标准差（对 RGB 三通道取均值）
3. std < 15 → simple, 15 ≤ std < 50 → gradient, std ≥ 50 → complex

**填充策略**:
- **Simple**: 计算背景区域的中值颜色，用该颜色填充所有文字区域
- **Gradient**: 对每个文字区域的每一行，取该行左右两侧背景像素做线性插值
- **Complex**: 白色填充（回退策略）

## 014: 图片提取（嵌入资源优先）

**Category:** New | **Priority:** High | **Status:** InProgress | **Started:** 2026-04-08

### 描述

实现 Image Extractor 阶段，从 PDF 中提取图片资源作为独立 ImageBlock 放入 SlideContent。在可编辑 PPTX 中以独立图片元素放置（而非保留在背景截图中）。优先使用 PDF 嵌入资源（高分辨率），回退到页面截图裁切。

### 设计要点

- 参考 DD §2.5 Image Extractor
- 两级提取策略：
  - **优先**: 匹配 PDF 嵌入资源（IoU > 0.5），使用原始分辨率图片
  - **回退**: 从页面渲染图中裁切 IMAGE 标签区域
- 输入：PageImage（含 embedded_images）+ LayoutResult
- 输出：`tuple[ExtractedImage, ...]`
- 数据模型已就绪：EmbeddedResource、ExtractedImage、ImageBlock

### 3. 技术方案

**依赖**: numpy, pillow, PyMuPDF（已安装）

**核心函数**:
```python
# parser.py — 嵌入资源提取
def _extract_embedded_resources(
    page: fitz.Page,
    doc: fitz.Document,
    dpi: int,
) -> tuple[EmbeddedResource, ...]:
    """Extract embedded image resources from a PDF page."""

# image.py — 图片提取阶段
def _filter_image_regions(
    regions: tuple[LayoutRegion, ...],
) -> tuple[LayoutRegion, ...]:
    """Filter layout regions to IMAGE label only."""

def _compute_iou(a: BoundingBox, b: BoundingBox) -> float:
    """Compute Intersection over Union of two bboxes."""

def _match_embedded(
    region: LayoutRegion,
    resources: tuple[EmbeddedResource, ...],
    iou_threshold: float = 0.5,
) -> EmbeddedResource | None:
    """Find best matching embedded resource for a layout region."""

def _crop_image(image: np.ndarray, bbox: BoundingBox) -> np.ndarray:
    """Crop image region with bbox clamping."""

def extract_images(
    page_image: PageImage,
    layout_result: LayoutResult,
) -> tuple[ExtractedImage, ...]:
    """Extract images from IMAGE-labeled regions."""
```

**匹配算法**:
1. 从 LayoutResult 过滤 IMAGE 标签区域
2. 对每个 IMAGE 区域，计算与所有 EmbeddedResource 的 IoU
3. IoU > 0.5 且最大者匹配成功（source="embedded"）
4. 无匹配则从 page_image.image 裁切（source="cropped"）

**嵌入资源过滤**:
- 跳过面积 < 100px² 的图片（装饰线条）
- 跳过宽高比极端的图片（> 50 或 < 0.02）
- 去重：同一 xref 多次出现取第一个
