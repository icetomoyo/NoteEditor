## 015: 字体映射表

**Category:** New | **Priority:** Medium | **Status:** Completed | **Started:** 2026-04-09

### 描述

实现 Font Matcher 阶段，使用字体映射表（font_map.yaml）为文本区域匹配字体。替换 builder 中的硬编码 Arial fallback，使可编辑 PPTX 的文本使用更准确的字体。支持字体文件存在性检查和 system_fallback 降级。

### 设计要点

- 参考 DD §2.5 Font Matcher
- 字体映射表 `fonts/font_map.yaml` 已就绪，含 3 种 label 映射
- 匹配策略：label → font_name → 检查 font_file 是否存在 → fallback
- 输入：LayoutResult + fonts_dir
- 输出：`tuple[FontMatch, ...]`
- 数据模型 FontMatch 已就绪

### 3. 技术方案

**依赖**: pyyaml（需添加）, pathlib（标准库）

**核心函数**:
```python
def _load_font_map(fonts_dir: Path) -> dict[str, dict[str, str]]:
    """Load font_map.yaml from fonts directory."""

def _match_font(
    label: RegionLabel,
    font_map: dict[str, dict[str, str]],
    fonts_dir: Path,
) -> FontMatch:
    """Match a font for a single region label."""

def match_fonts(
    layout_result: LayoutResult,
    fonts_dir: Path,
) -> tuple[FontMatch, ...]:
    """Match fonts for all text regions in a layout."""
```

**匹配算法**:
1. 加载 fonts/font_map.yaml
2. 对每个文本区域（label ∈ {TITLE, BODY_TEXT, EQUATION, CODE_BLOCK}）:
   a. 以 label 值为键查映射表
   b. 找到 → 检查 font_file 是否存在于 fonts_dir:
      - 存在: font_path=文件路径, is_fallback=False
      - 不存在: 使用 system_fallback, is_fallback=True
   c. 未找到映射: 使用 Arial 默认, is_fallback=True
3. 返回 FontMatch 列表
