## 016: 字号颜色估计

**Category:** New | **Priority:** Medium | **Status:** Completed | **Started:** 2026-04-09

### 描述

实现 Text Style Estimator 阶段，从页面图像中估算文本区域的字号和颜色。替换 builder 中的固定字号公式和缺失的颜色设置，使可编辑 PPTX 的文本样式更接近原始 PDF。

### 设计要点

- 参考 DD §2.7 Builder — 字号估算和颜色采样
- 字号：沿用 bbox 高度公式 `bbox_height_px * (72 / dpi) * 0.8`，从 builder 移到独立阶段
- 颜色：从 PageImage.image 中采样 bbox 顶部中间区域像素的众数颜色
- 输入：PageImage + LayoutResult
- 输出：`tuple[TextStyle, ...]`
- 数据模型 TextStyle 新增

### 3. 技术方案

**依赖**: numpy（已安装）, PIL（已安装）

**数据模型**:
```python
@dataclass(frozen=True)
class TextStyle:
    """Estimated text style for a region."""
    region_id: str
    font_size_pt: int
    font_color_rgb: tuple[int, int, int]
```

**核心函数**:
```python
def _estimate_font_size(bbox_height_px: float, dpi: int) -> int:
    """Estimate font size from bbox height."""

def _sample_font_color(
    image: np.ndarray,
    bbox: BoundingBox,
) -> tuple[int, int, int]:
    """Sample dominant text color from page image at bbox region."""

def estimate_styles(
    page_image: PageImage,
    layout_result: LayoutResult,
) -> tuple[TextStyle, ...]:
    """Estimate text styles for all text regions."""
```

**颜色采样算法**:
1. 取 bbox 对应区域（顶部 1/3 范围，中间 80% 宽度）
2. 对区域内像素做 k-means 或简单统计
3. 返回最常见的深色像素 RGB（排除接近白色/背景色）
4. 回退策略：无法采样时返回黑色 (0, 0, 0)

### 4. 接口契约
