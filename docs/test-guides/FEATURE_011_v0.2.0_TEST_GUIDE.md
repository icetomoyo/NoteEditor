# Feature 011 - 可编辑文本框放置 人工测试指南

**版本**: v0.2.0
**日期**: 2026-04-08
**前置条件**: Features 009 (布局检测) 和 010 (OCR) 已完成

---

## 前置条件

1. 准备一个 NotebookLM 导出的 PDF 文件（含文字和图片区域）
2. 确保 PP-DocLayout-V3 模型已下载到 `~/.noteeditor/models/`
3. 确保 ZhipuAI API Key 已配置（如果使用 API 模式）

## 测试用例

### TC-001: 基础可编辑 PPTX 输出

**步骤**:
1. 运行 pipeline（需先完成 Feature 012 集成后）或直接调用 `build_editable_pptx()`
2. 用 PowerPoint/WPS 打开输出的 PPTX 文件
3. 点击文字区域

**预期结果**:
- 文字区域可直接选中、编辑、删除
- 文本框有白色填充背景
- 图片区域没有文本框覆盖

### TC-002: 文本框位置对齐

**步骤**:
1. 用 PowerPoint 打开可编辑 PPTX
2. 对比原始 PDF 和 PPTX 中的文字位置
3. 检查标题和正文的对齐

**预期结果**:
- 文本框位置与原始 PDF 文字位置基本对齐
- 标题文本居中对齐
- 正文文本左对齐

### TC-003: 字号视觉匹配

**步骤**:
1. 在 PowerPoint 中选中文本框
2. 查看字号（Home 标签页显示）
3. 对比原始 PDF 中的视觉大小

**预期结果**:
- 字号与原始 PDF 视觉大小大致匹配
- 标题字号明显大于正文字号
- 字体为 Arial

### TC-004: 多页文档

**步骤**:
1. 使用多页 PDF 输入（3+ 页）
2. 检查每页的文本框放置

**预期结果**:
- 每页都有正确的文本框
- 页码顺序正确
- 幻灯片总数与 PDF 页数一致

### TC-005: 失败页面回退

**步骤**:
1. 构造一个 layout 检测失败的场景（如传入 status="fallback" 的 SlideContent）
2. 检查输出的 PPTX

**预期结果**:
- 回退页面仅包含背景截图，无文本框
- 不影响其他成功页面

### TC-006: 空文档

**步骤**:
1. 传入空的 pages tuple
2. 检查输出的 PPTX

**预期结果**:
- 输出有效的 PPTX 文件
- 0 张幻灯片

---

## 边界测试

### BT-001: 仅图片页面
- 输入只有图片区域（无文字区域）的页面
- 预期：无文本框，仅背景截图

### BT-002: 大量文本区域
- 输入包含 20+ 个文字区域的页面
- 预期：所有文本框正确放置，不重叠

### BT-003: 公式区域
- 输入包含 LaTeX 公式的 OCR 结果
- 预期：公式以文本形式显示（LaTeX 源码）

### BT-004: 极小文本区域
- 输入高度很小的 bbox（如 5px）
- 预期：字号最小为 1pt，不崩溃
